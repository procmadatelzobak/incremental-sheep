# Issue #61 — Pages deploy workflow nesmí běžet z libovolných větví

issue: 61

## Kontext

Privileged Pages workflow (`pages: write`, `id-token: write`) se spouštěl na
`push` do libovolné branche (`branches: ['**']`). Branch mohla protlačit
vlastní upravenou workflow definici do privileged deploye bez review
(P1 z Codex review na PR #56).

## Cíl

- `push` trigger běží jen na trusted `main`.
- `pull_request_target` (PR na main), ruční `workflow_dispatch` a `schedule`
  zůstávají — branch preview je best-effort z těchto trusted rebuildů.
- Nikde v configu není `'**'` wildcard pro push branche.
- Acceptance test v `test/pages-workflow.test.mjs` tuhle invariantu ověřuje
  (ne naopak — původní test omylem tvrdil zranitelné chování).

## Akceptační testy

```yaml acceptance
- name: celá testovací sada je zelená
  run: npm test
- name: push trigger běží na main
  file: .github/workflows/pages-previews.yml
  must_match: "push:\s*\n\s*branches:\s*\[[^\]]*\bmain\b[^\]]*\]"
- name: žádný wildcard push trigger
  file: .github/workflows/pages-previews.yml
  must_not_match: "branches:\s*\[[^\]]*\*\*[^\]]*\]"
- name: PR a schedule trigger zůstávají
  file: .github/workflows/pages-previews.yml
  must_match: "pull_request_target:[\s\S]*schedule:"
- name: acceptance test invariantu existuje
  file: test/pages-workflow.test.mjs
  must_match: "#61"
```

## Poznámky pro příště (z tohoto běhu)

Test byl součást specifikace a původně kódoval *zranitelné* chování
(`branches: ['**']`). Bylo nutné ho **obrátit**, ne jen obejít. Lekce:
než začneš implementovat „do zelena", zkontroluj, že akceptační test
opravdu kóduje záměr issue.
